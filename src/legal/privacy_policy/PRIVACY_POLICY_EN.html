2. Information We Collect

2.1 Information You Provide

Account Information: - Email address (for authentication via Supabase) - Password (encrypted and securely stored)

Vehicle Information: - Vehicle make, model, year - VIN number - Odometer readings - Maintenance history and schedules - Notes and custom data you add

Trip Data: - Trip start/end times and locations - Distance traveled - Route information - Trip notes and categories

2.2 Automatically Collected Information

Location Data: - GPS coordinates during active navigation or trip tracking - Location data is stored locally on your device and synced to Supabase for backup - We do NOT share precise GPS coordinates with third parties except for navigation services (Valhalla, Nominatim, Overpass (self-hosted; speed limit data, POI, Nominatim search support), Mapbox (fallback)) temporarily during active navigation - Location permission is required for navigation and trip tracking features

Device Information: - Device model and manufacturer - Operating system version (Android/iOS) - App version - Device language and region settings

Usage Analytics (Anonymous): - Screen views and feature usage - Service usage statistics (Nominatim, Valhalla, Nominatim, Overpass (self-hosted; speed limit data, POI, Nominatim search support), Mapbox (fallback)) - App performance metrics and crash reports - Error logs and debugging information

Important: We do NOT collect or store precise GPS coordinates for analytics purposes. Location data is only used for core app functionality (navigation, trip tracking) and is never shared with analytics services.

2.3 Third-Party Services Integration

OpenAI Integration: - Vehicle make, model, year, and engine specifications (VIN-derived data) - Mileage information - Used ONLY for AI-powered maintenance interval recommendations - We do NOT send: personal information, trip routes, or precise locations to OpenAI

3. How We Use Your Information

We use collected information for the following purposes:

3.1 Core Functionality (Free)

  • Provide vehicle management and maintenance tracking
  • Track and save your trips manually
  • Generate personalized maintenance schedules using AI analysis
  • Sync your data across devices
  • Send maintenance reminders via push notifications

3.2 Premium Features

  • Enable GPS navigation with real-time directions
  • Track and save your trips automatically (background tracking)
  • Provide component lifespan predictions based on your vehicle specifications

3.3 Service Improvement

  • Analyze app usage patterns (anonymously) to improve user experience
  • Monitor service performance (Nominatim, Valhalla, Nominatim, Overpass (self-hosted; speed limit data, POI, Nominatim search support), Mapbox (fallback))
  • Detect and fix bugs, crashes, and errors
  • Optimize routing algorithms and navigation accuracy

3.4 Communication

  • Send you maintenance reminders and notifications
  • Respond to your support requests
  • Inform you about important app updates or changes

4. Data Storage and Security

4.1 Where Your Data is Stored

Local Storage: - All trip data, vehicle information, and maintenance records are stored locally on your device using encrypted SQLite database

Cloud Backup (Supabase): - Your data is backed up to Supabase cloud servers for synchronization - Supabase is compliant with GDPR and uses encryption in transit and at rest - Servers are located in: North America, Central America, and Europe

4.2 Data Security Measures

  • End-to-end encryption for data transmission
  • Encrypted local storage using industry-standard encryption
  • Secure authentication with Supabase Auth
  • Regular security audits and updates
  • No third-party advertising networks with access to your data

4.3 Data Retention

  • Active Users: Your data is retained as long as you maintain an active account
  • Deleted Accounts: All personal data is deleted immediately upon account deletion
  • Trip History: Stored indefinitely unless you manually delete trips
  • Analytics Data: Anonymous analytics data is retained for 26 months (Firebase default)
